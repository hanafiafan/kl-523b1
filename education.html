Dokumentasi & Edukasi K-Means Clustering

Memahami konsep-konsep dasar algoritma K-Means dalam Data Mining

Apa itu K-Means Clustering?

K-Means adalah salah satu algoritma unsupervised learning yang paling populer dalam Data Mining. Algoritma ini digunakan untuk mengelompokkan data ke dalam K kelompok (cluster) berdasarkan kemiripan.

Tujuannya adalah membuat data dalam satu cluster saling mirip (homogen), sementara data antar cluster berbeda (heterogen). Misalnya, kita bisa mengelompokkan pelanggan ke dalam segmen seperti: "Pelanggan Muda dengan Income Rendah", "Pelanggan Dewasa dengan Income Tinggi", dll.

Apa itu Centroid?

Centroid adalah titik pusat dari setiap cluster. Bayangkan centroid sebagai "perwakilan" atau "rata-rata" dari semua anggota cluster tersebut.

Jika kita mengelompokkan pelanggan berdasarkan Age dan Income, maka centroid adalah titik (rata-rata Age, rata-rata Income) dari semua pelanggan dalam cluster tersebut.

Pada awal algoritma, centroid dipilih secara acak dari data. Seiring iterasi berjalan, centroid akan berpindah ke posisi yang lebih optimal.

Apa itu Euclidean Distance?

Euclidean Distance adalah cara mengukur jarak antara dua titik. Ini adalah jarak "garis lurus" yang biasa kita gunakan sehari-hari.

d = √( (x₁ - x₂)² + (y₁ - y₂)² )

Contoh: Jika pelanggan A memiliki Age=30 dan Income=50, sedangkan Centroid C1 memiliki Age=35 dan Income=60, maka:

d = √( (30-35)² + (50-60)² )
d = √( (-5)² + (-10)² )
d = √( 25 + 100 )
d = √125
d = 11.18

Jarak ini digunakan untuk menentukan pelanggan lebih dekat ke cluster mana. Pelanggan akan masuk ke cluster dengan centroid terdekat (jarak terkecil).

Mengapa Iterasi Dibutuhkan?

K-Means bekerja secara iteratif (berulang). Berikut langkah-langkahnya:

  1. Inisialisasi: Pilih K titik acak sebagai centroid awal
  2. Assignment: Hitung jarak setiap data ke semua centroid, lalu masukkan data ke cluster terdekat
  3. Update: Hitung centroid baru = rata-rata semua anggota cluster
  4. Cek Konvergensi: Jika centroid tidak berubah → selesai. Jika masih berubah → kembali ke langkah 2

Iterasi dibutuhkan karena pada awalnya centroid dipilih secara acak, sehingga posisinya belum optimal. Dengan setiap iterasi, centroid bergeser mendekati pusat sebenarnya dari cluster, sehingga pengelompokan menjadi semakin baik.

Algoritma dikatakan konvergen ketika centroid tidak berubah lagi — artinya semua data sudah berada di cluster yang paling cocok.

Apa itu Elbow Method?

Salah satu tantangan K-Means adalah menentukan nilai K (jumlah cluster) yang optimal. Elbow Method membantu kita menemukan K terbaik.

Caranya:

  1. Jalankan K-Means untuk berbagai nilai K (misalnya K=2, 3, 4, ..., 10)
  2. Hitung SSE (Sum of Squared Errors) untuk setiap K — ini adalah total jarak kuadrat dari setiap data ke centroid cluster-nya
  3. Plot grafik SSE vs K
  4. Cari titik "siku" (elbow) — titik di mana penurunan SSE mulai melambat secara signifikan

Titik siku tersebut adalah nilai K optimal. Menambah cluster setelah titik itu tidak memberikan peningkatan signifikan.
